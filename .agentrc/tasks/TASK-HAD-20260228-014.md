## Description

Implement the `hadron` CLI. All commands communicate with the daemon via REST API. Use `cobra` for subcommand structure.

## Instructions

`cmd/hadron/main.go` with cobra root command. Global flag: `--addr` (daemon base URL, default `http://127.0.0.1:8095`).

**Commands:**

```
hadron run <blueprint-path>
  --input key=value (repeatable)
  --workspace string (default "default")
  --dry-run
  # Enqueues run via POST /v1/runs, polls events, streams to stdout

hadron validate <blueprint-path>
  # Sends to POST /v1/blueprints/validate, prints result

hadron blueprint list [--dir path]
  # Lists *.yaml/*.json files in dir (local fs, not via daemon)
  # Validates each and shows name, status (valid/invalid)

hadron blueprint show <path>
  # Prints parsed blueprint summary (name, inputs, sections, tasks)

hadron schedule list [--workspace string]
hadron schedule create --blueprint <path> --cron <expr> [--name str] [--workspace str]
hadron schedule enable <id>
hadron schedule disable <id>
hadron schedule delete <id>

hadron pipeline run <pipeline-path> [--workspace str]

hadron workspace list
hadron workspace create <name>

hadron daemon status [--addr url]
  # GET /v1/health and prints status
```

**`hadron run` streaming:**
- POST /v1/runs → get run ID
- Poll GET /v1/runs/:id/events with last cursor until run status=done/failed
- Print each event line as it arrives
- Exit with code 0 (done) or 1 (failed)

## Acceptance

- `go build ./cmd/hadron` passes
- `hadron daemon status` prints daemon health or "daemon not reachable"
- `hadron validate examples/hello-hadron.yaml` prints "valid" or error message
- `hadron run examples/hello-hadron.yaml` streams output and exits correctly
- `hadron schedule create --blueprint x.yaml --cron "0 * * * *"` creates schedule
- All commands show `--help` text
