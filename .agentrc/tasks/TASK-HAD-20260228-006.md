## Description

Port the settings package from wails-starter. Provides safety rules (command allowlist/denylist, path validation, sudo blocking) and UI/execution preferences. This is a dependency of the execution manager.

## Instructions

- Port `reference-only/nanite-wails-starter/backend/settings/settings.go` → `hadron/internal/settings/settings.go`
- Update module path
- Extend `ExecutionSettings` with a `Workers` field (default 3) to replace the hardcoded worker count
- Keep: `AllowedCommands`, `DeniedCommands`, `AllowedDirs`, `DeniedDirs`, `DefaultTimeout`, `MaxConcurrentJobs`
- Keep: `SafetySettings` (RequireConfirmation, DryRunByDefault, BlockSudo, SandboxMode)
- Remove: `UISettings` (UI concerns belong in Phase 2, not the daemon)
- Default denied commands should include: `rm -rf /`, `dd`, `mkfs`, `format`, `shutdown`, `reboot`
- `ValidateCommand` and `ValidatePath` methods stay unchanged
- `Load(dataDir)` and `Save(dataDir)` stay unchanged

## Acceptance

- Settings loads from `settings.json` in data dir, falls back to defaults if absent
- `ValidateCommand("sudo rm -rf /")` returns error when BlockSudo=true
- `ValidatePath("/System/Library/foo")` returns error (denied dir)
- `go build ./internal/settings/...` passes
