## Description

Implement the Hadron blueprint spec v0.4 Go package. This is the most critical task in Sprint A — the spec model is what all other packages depend on.

## Instructions

Build `hadron/internal/blueprint/blueprint.go` implementing the full v0.4 spec:

**Structs to implement** (see `artifacts/plan/01-blueprint-spec-v04.md` for full field list):
- `Blueprint` — top-level
- `BlueprintInfo` — `blueprint:` meta block
- `Project` — project block
- `Input` — typed input with all v0.4 constraints (label, min/max, minLength/maxLength, pattern, items_type)
- `Packages` — rich ecosystem (Composer, NPM, Pip, Brew, Go sub-structs with shorthand unmarshalers)
- `Git`, `Stubs`, `Tools` — config blocks
- `Import` — with alias + with map
- `Hooks`, `Hook` — blueprint-level lifecycle hooks
- `Section` — `{section string, tasks []Task}`
- `Task` — full v0.4 task (cmd, run, call, if, condition, dir, env, retry, retry_delay_seconds, timeout_seconds, continue_on_error, enabled, with, on_success, on_fail)
- `ActionHook` — `{type string, value string}`

**Functions to implement**:
- `ParseFile(path string) (*Blueprint, error)` — reads file, uses specparse, validates
- `ParseBytes(data []byte) (*Blueprint, error)` — uses specparse with empty path, validates
- `Validate(bp *Blueprint) error` — full validation (see spec rules in `01-blueprint-spec-v04.md`)
- `NormalizeInputs(bp *Blueprint, values map[string]any) (map[string]any, error)`
- `BuildTemplateContext(bp *Blueprint, blueprintPath, workspaceID string, inputs map[string]any) map[string]any`
- `RenderForExecution(bp *Blueprint, ctx map[string]any) (*Blueprint, error)` — renders all template strings in all fields
- `LoadWithImports(path string) (*Blueprint, error)` — recursively resolves imports, merges steps

**Template functions** (all from v0.2 source plus `json`):
- `upper`, `lower`, `trim`, `replace`, `split`, `join`, `basename`, `dirname`, `ext`, `env`, `readFile` (1MB guard), `default`, `ternary`, `json`

**Compatibility aliases** (accept but normalize):
- `condition` → `if`
- `retryDelay` / `retry_delay` → `retry_delay_seconds`
- `continueOnError` → `continue_on_error`
- `composer: [string]` shorthand → `composer.require`

## Acceptance

- All structs match v0.4 spec
- `Validate` catches: missing name, empty steps, invalid input type, duplicate input name, invalid pattern regex, min > max, duplicate import alias, task with neither cmd nor call
- `Validate` accepts: v0.2 blueprint with `condition:` and camelCase compat fields
- `RenderForExecution` renders Go templates in cmd, dir, name, env values, if, call
- `NormalizeInputs` fills defaults, rejects missing required, rejects unknown inputs, validates typed constraints
