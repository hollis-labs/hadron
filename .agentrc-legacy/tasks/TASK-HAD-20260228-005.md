## Description

Write a comprehensive test suite for the v0.4 blueprint package. Must cover validation rules, compat aliases, template rendering, input normalization, and import resolution.

## Instructions

Create `hadron/internal/blueprint/blueprint_test.go` covering:

**Parse tests:**
- Parse valid YAML blueprint (v0.4)
- Parse valid JSONC blueprint with comments and trailing commas
- Parse v0.2 blueprint (camelCase compat, sections structure, packages shorthand)
- Parse blueprint with all field types populated

**Validation tests (one test per rule):**
- Missing `blueprint.name` → error
- Empty `steps` → error
- Section with no tasks → error
- Task with neither `cmd` nor `call` → error
- Duplicate input name → error
- Invalid input type → error
- Invalid regex `pattern` → error
- `min` > `max` → error
- `min_length` > `max_length` → error
- Duplicate import alias → error
- `retry` < 0 → error
- Valid blueprint with all optional fields absent → no error

**Template rendering tests:**
- `RenderForExecution` renders `{{ .inputs.name }}` in cmd
- `RenderForExecution` renders `{{ .project.vars.app_name }}` in dir
- `RenderForExecution` renders `{{ env "HOME" }}` in env value
- `readFile` size guard returns error for >1MB

**Input normalization tests:**
- Required input missing → error
- Unknown input → error
- Default applied when input absent
- Enum validation (value not in enum → error)
- Pattern mismatch → error
- min/max violations → error

**Compat tests:**
- `condition:` field parsed as `if`
- `continueOnError:` parsed as `continue_on_error`
- `retryDelay:` parsed as `retry_delay_seconds`
- `composer: [string]` shorthand normalized

**Integration:**
- Load `../../reference-only/nanite-spec-v0.2/reference-blueprint.yaml` → must parse and validate without error

## Acceptance

- `go test ./internal/blueprint/...` passes
- All cases above have explicit test functions
- Reference blueprint from v0.2 validates successfully
