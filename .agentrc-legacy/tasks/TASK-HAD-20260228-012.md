## Description

Implement the REST API server for `hadrond`. All endpoints documented in `artifacts/plan/02-architecture.md`.

## Instructions

Create `hadron/internal/api/server.go`:

- Use `net/http` stdlib (no external router needed for this surface)
- JSON request/response throughout (`Content-Type: application/json`)
- All errors: `{"error": "message"}` with appropriate HTTP status
- All list responses: `{"items": [...], "next_cursor": "..."|null}`

**Endpoints to implement** (see architecture doc for full list):

Health:
- `GET /v1/health` → `{"status":"ok","version":"0.4.0"}`

Workspaces:
- `GET /v1/workspaces`
- `POST /v1/workspaces` body: `{"id":"","name":""}`
- `GET /v1/workspaces/:id`

Runs:
- `GET /v1/runs?workspace_id=&limit=&cursor=`
- `POST /v1/runs` body: `{"workspace_id":"","blueprint_path":"","inputs":{}}`
- `GET /v1/runs/:id`
- `DELETE /v1/runs/:id` (cancel if running)
- `GET /v1/runs/:id/events?limit=&cursor=`

Schedules:
- `GET /v1/schedules?workspace_id=`
- `POST /v1/schedules` body: `{"workspace_id":"","name":"","blueprint_path":"","cron_expr":"","enabled":true}`
- `GET /v1/schedules/:id`
- `PATCH /v1/schedules/:id` body: `{"enabled":bool,"next_run_at":"RFC3339"|null}`
- `DELETE /v1/schedules/:id`

Pipelines:
- `GET /v1/pipelines?workspace_id=&limit=`
- `POST /v1/pipelines` body: `{"workspace_id":"","pipeline_path":"","inputs":{}}`
- `GET /v1/pipelines/:id`
- `GET /v1/pipelines/:id/stages`

Blueprints:
- `POST /v1/blueprints/validate` body: raw YAML/JSON bytes → `{"valid":true}` or `{"valid":false,"error":"..."}`

## Acceptance

- `go build ./internal/api/...` passes
- `GET /v1/health` returns 200
- `POST /v1/runs` enqueues a run and returns the run record
- `GET /v1/runs/:id/events` returns persisted events for a completed run
- All 404s return `{"error":"not found"}` not HTML
