## Description

Smoke tests for the REST API and MCP adapter. Confirms the full external surface works before hardening sprint.

## Instructions

`hadron/internal/api/server_test.go`:
- Spin up `api.Server` with a temp SQLite DB in test
- `GET /v1/health` → 200 `{"status":"ok"}`
- `POST /v1/workspaces` → 201, workspace in list
- `POST /v1/runs` with a simple `echo` blueprint → 202, run created
- `GET /v1/runs/:id` → run record returned
- `POST /v1/blueprints/validate` with valid YAML → `{"valid":true}`
- `POST /v1/blueprints/validate` with invalid YAML → `{"valid":false,"error":"..."}`
- `POST /v1/schedules` → 201, schedule created
- `PATCH /v1/schedules/:id` enable/disable → 200
- Unknown route → 404 `{"error":"not found"}`

`hadron/internal/mcpadapter/integration_test.go`:
- Create Adapter with temp store
- `hadron_health` tool returns ok
- `hadron_blueprint_validate` with valid content returns valid=true
- `hadron_blueprint_validate` with invalid content returns valid=false with error

## Acceptance

- All smoke tests pass
- No test uses real filesystem blueprints (all inline test fixtures)
- Tests run in < 5 seconds total
