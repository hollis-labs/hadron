## Description

Full end-to-end tests that start an actual `hadrond` daemon in a subprocess, run CLI commands against it, and verify results. This is the final quality gate before Sprint D is considered done.

## Instructions

Create `hadron/test/e2e/e2e_test.go`:

**Setup:** `TestMain` starts `hadrond serve` on a random port with a temp DB, waits for health, runs all tests, shuts down daemon.

**Test cases:**

1. **Health check** — `hadron daemon status` exits 0, output contains "ok"

2. **Run hello-hadron** — `hadron run examples/hello-hadron.yaml` exits 0, output contains expected echo text

3. **Validate valid blueprint** — `hadron validate examples/hello-hadron.yaml` exits 0, prints "valid"

4. **Validate invalid blueprint** — `hadron validate` on a malformed file exits 1, prints error

5. **Blueprint lint** — `hadron lint examples/` exits 0 (all examples valid)

6. **Schedule lifecycle** — create schedule, list shows it, disable it, list shows disabled

7. **Workspace** — `hadron workspace create test-ws`, list shows it

8. **Run with inputs** — `hadron run examples/parameterized.yaml --input app_name=e2e-test` exits 0

## Acceptance

- All 8 e2e tests pass
- Tests are tagged `//go:build e2e` so they don't run with `go test ./...` by default
- Run with: `go test -tags e2e ./test/e2e/...`
- `make e2e` target added to Makefile

## Notes

- Use `os/exec` to run `hadrond` and `hadron` binaries (must be pre-built)
- Use `t.TempDir()` for all data dirs
- Port randomization: use `:0` or find a free port to avoid conflicts
