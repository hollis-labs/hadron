## Description

Implement the `hadrond` daemon binary. Wires together all internal packages and starts the HTTP server and scheduler.

## Instructions

`cmd/hadrond/main.go` with two subcommands:

**`hadrond serve` (default):**
- Parse flags: `--addr`, `--db`, `--logs`, `--data`
- Call `config.Default()`, apply flag overrides, call `config.Ensure()`
- Open `persistence.Store`
- Load `settings.Settings`
- Create `execution.Manager`
- Create `scheduler.Engine`, wire to Manager
- Start scheduler engine
- Start HTTP server (api.Server) on addr
- Handle SIGINT/SIGTERM: stop scheduler, drain active jobs (with timeout), close store
- Log startup info: listening addr, db path, version

**`hadrond mcp`:**
- Parse flags: `--db`, `--logs`, `--data`, `--token`, `--token-scopes`
- Wire persistence + execution + scheduler (same as serve)
- Start MCP adapter in stdio mode
- No HTTP server

## Acceptance

- `go build ./cmd/hadrond` passes
- `./hadrond serve --addr 127.0.0.1:8095` starts and logs startup
- `curl http://127.0.0.1:8095/v1/health` returns 200
- `./hadrond serve` exits cleanly on SIGINT
- `./hadrond mcp --token dev` starts MCP stdio mode without panicking
