## Description

Wire structured telemetry logging through the full execution stack. Every run should produce a structured JSONL log file alongside its SQLite events.

## Instructions

Port `reference-only/nanite-wails-starter/backend/telemetry/telemetry.go` → `hadron/internal/telemetry/telemetry.go`:
- Update module path
- Keep: Logger, Event, JobMetadata, JobCompletionStats, TaskExecution structs
- Keep: LogJobStart, LogJobEnd, LogTaskExecution, Log methods
- Wire into execution.Manager (already planned in TASK-007 notes, formalize here)

**Log output format:** JSONL file per run at `{logs_dir}/{run_id}.jsonl`

**Events to log:**
- Job start (blueprint path, name, version, hostname, hadron version)
- Each task start
- Each task completion (duration, exit code, output snippet)
- Each task failure (error, retry count)
- Job end (state, duration, total/completed/failed/retried/timed_out/skipped counts)
- Security blocks (command denied by safety settings)

**Daemon startup log** (stdout, structured):
- `{"level":"info","msg":"hadron daemon starting","addr":"...","db":"...","version":"0.4.0"}`

**Settings:** add `Telemetry.Enabled bool` (default true) and `Telemetry.RetainDays int` (default 30)

## Acceptance

- After running `hadron run examples/hello-hadron.yaml`, a JSONL log file exists in logs dir
- Log file contains job start, task events, and job end entries
- `telemetry.Enabled=false` in settings suppresses log file creation (SQLite events still work)
- Daemon startup logs structured JSON to stdout
