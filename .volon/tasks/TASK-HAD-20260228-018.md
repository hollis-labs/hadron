## Description

Add `hadron lint` and `hadron fmt` CLI commands for blueprint quality checks and auto-formatting. Useful for CI pipelines and agent workflows.

## Instructions

**`hadron lint <path|dir>`:**
- If path is a file: validate and report all errors with line hints where possible
- If path is a directory: scan recursively for `*.yaml`, `*.yml`, `*.json`, `*.jsonc` and lint each
- Output format: `file:section — error message`
- Exit code: 0 if all valid, 1 if any invalid
- Flag: `--json` to output machine-readable `[{"file":"","errors":[]}]`

**`hadron fmt <path>`:**
- Parse the blueprint (skip validation)
- Re-serialize to canonical YAML format (sorted keys, consistent indentation)
- Flag: `--check` — exit 1 if file would change (for CI)
- Flag: `--write` — write normalized YAML back to file (default: print to stdout)

**Compat alias normalization in fmt:**
- `condition:` → `if:`
- `continueOnError:` → `continue_on_error:`
- `retryDelay:` → `retry_delay_seconds:`
- Normalize package shorthands

## Acceptance

- `hadron lint examples/` validates all example blueprints and exits 0
- `hadron lint` on a broken blueprint exits 1 and shows error
- `hadron lint --json` outputs parseable JSON
- `hadron fmt --check examples/hello-hadron.yaml` exits 0 if already canonical
- `hadron fmt examples/hello-hadron.yaml` prints canonical YAML to stdout
