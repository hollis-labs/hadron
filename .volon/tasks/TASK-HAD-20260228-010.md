## Description

Write an integration test that exercises the full execution stack: parse blueprint → normalize inputs → render → enqueue → execute via PTY → events persisted to SQLite → run marked done.

## Instructions

Create `hadron/internal/execution/integration_test.go`:

**Test 1: hello-world blueprint**
- Blueprint: one section, one task — `echo "hello hadron"`
- Enqueue run → wait for completion → assert run status=done
- Assert at least one run_event with event_type="log" containing "hello hadron"

**Test 2: retry on failure**
- Blueprint: task runs `exit 1`, retry=2
- Assert run status=failed
- Assert 3 run_events with event_type="log" showing retry attempts

**Test 3: condition skip**
- Blueprint: task with `if: "false"` (literal)
- Assert task was skipped (run_event with event_type="skipped" or task not in events)

**Test 4: dry run**
- Blueprint: task runs `touch /tmp/hadron-test-file`
- Run with DryRun=true
- Assert file was NOT created
- Assert run completed (dry runs don't fail)

**Test 5: on_fail hook**
- Blueprint: task `exit 1` with `on_fail: [{type: cmd, value: "echo failed-hook"}]`
- Assert "failed-hook" appears in run events

Use an in-memory or temp-dir SQLite for all tests.

## Acceptance

- All 5 integration tests pass
- Tests are deterministic (no timing-based races)
- Temp files cleaned up after each test

## Notes

- Use `t.TempDir()` for SQLite DB path
- Worker count = 1 for deterministic ordering
- Add a reasonable timeout to the test context (30s max per test)
