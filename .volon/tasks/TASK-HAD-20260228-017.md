## Description

Create a reference set of example blueprints that collectively demonstrate all major v0.4 features. These serve as user documentation, integration test fixtures, and dogfood for Volon/Mentat.

## Instructions

Create `hadron/examples/`:

**`hello-hadron.yaml`** — minimal v0.4 blueprint
- 1 section, 2 tasks (echo commands)
- Shows: version, blueprint meta, basic step structure
- Should run successfully on any macOS/Linux machine with no dependencies

**`laravel-app.yaml`** — Laravel project scaffold
- Port/adapt from `reference-only/nanite-spec-v0.2/reference-blueprint.yaml`
- Sections: Bootstrap, PHP & Laravel, Frontend, Project Hygiene
- Uses: project block, packages (composer+npm), git block, stubs block, inputs (app_name), template functions
- Shows: `{{ .inputs.app_name }}`, `{{ .project.path }}`, `onSuccess`/`onFail` hooks

**`dev-cleanup.yaml`** — daily maintenance routine
- Sections: Temp Files, Logs, Build Artifacts
- Uses: `if:` conditions, `continue_on_error: true`, env vars, brew tools check
- Shows: shell conditions, safe failure handling

**`parameterized.yaml`** — typed input showcase
- Inputs: string with pattern, number with min/max, boolean, array, enum
- Shows: `{{ .inputs.name }}`, `{{ .inputs.count }}`, template ternary
- All inputs used in step commands

**`pipeline-demo/`** — 2-blueprint pipeline
- `setup.yaml` — install deps step
- `test.yaml` — run tests step
- `pipeline.yaml` — pipeline spec referencing both blueprints
- Shows: pipeline stages, stop_on_fail

**`hooks-demo.yaml`** — blueprint-level hooks + per-task hooks
- `hooks.before_run` — echo "starting"
- `hooks.after_run` — echo "finished"
- `hooks.on_error` — echo "something went wrong"
- Tasks with `on_success` and `on_fail` hooks of type cmd, error, step

## Acceptance

- `hadron validate examples/hello-hadron.yaml` → valid
- `hadron validate examples/laravel-app.yaml` → valid
- `hadron validate examples/parameterized.yaml` → valid
- `hadron validate examples/hooks-demo.yaml` → valid
- `hadron validate examples/pipeline-demo/pipeline.yaml` → valid
- `hadron run examples/hello-hadron.yaml` → executes and exits 0
- All examples have comments explaining the features they demonstrate
